Phone Directory (Hash Table)

Flask-based phone directory web application implementing hash table data structure for efficient contact management. Features include fast O(1) search, add, delete, and update operations with collision handling, persistent storage, and a clean, responsive web interface.

Duration: 3 weeks Domain: Data Structures & Web Dev Type: Web Application

Project Overview

Developed a web-based phone directory application that demonstrates practical implementation of hash table data structures. Built with Flask, the application provides a user-friendly interface for managing contacts with names, phone numbers, and email addresses. The core feature is a custom hash table implementation that ensures O(1) average time complexity for all operations. The hash function efficiently distributes contacts across buckets, and collision resolution is handled using chaining with linked lists. The application includes a responsive web interface for adding, searching, updating, and deleting contacts, along with features like bulk import/export and search filtering.

Key Features

  • Hash Table Implementation: Custom hash table with efficient hash function and bucket management.
  • Fast Search: O(1) average time complexity for contact lookups by name or phone number.
  • CRUD Operations: Add, search, update, and delete contacts with instant updates.
  • Collision Handling: Efficient chaining mechanism using linked lists for hash collisions.
  • Responsive Design: Mobile-friendly interface built with Bootstrap and custom CSS.
  • Import/Export: Bulk contact management with CSV import and export functionality.

Technologies & Stack

Technical Highlights

  • Custom Hash Function: Optimized hash function minimizing collisions with uniform distribution.
  • Collision Resolution: Separate chaining with linked lists for efficient conflict management.
  • Dynamic Resizing: Automatic table resizing and rehashing when load factor exceeds threshold.
  • Flask Routes: RESTful API endpoints for contact operations with proper HTTP methods.
  • Data Persistence: JSON-based storage for maintaining contacts between sessions.
  • Input Validation: Form validation for phone numbers, emails, and required fields.

The custom implementation handles collisions efficiently and provides fast contact management for up to 1000+ entries. The web interface offers an intuitive user experience with responsive design and comprehensive contact management features. This project showcases expertise in data structures, algorithm implementation, web development with Flask, and creating practical applications that solve real-world problems with optimal performance.
